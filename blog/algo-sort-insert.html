    # 插入排序

    2019.03.18

    整理扑克牌的过程完美诠释了插入排序的思想，即从底牌中接起一张牌跟手里的已排好序的牌对比，插入相应的位置，使整把牌有序。

    # 执行过程

    步骤：

    ①从第1个元素开始，依次取出与其前一个元素对比。若比其前一元素小，则再将待排元素依次和序列元素从右往左对比，对比过程中，若小于被比较序列的元素，则将序列后移一位。

    ②移位结束后，a[j]即是待排元素合适的位置。

    # js代码实现

    function insertSort(arr) {
    let n = arr.length;
    for(let i = 1; i < n; i++) {
        if(arr[i] < arr[i-1]) {
            let tmp = arr[i],
                j=0;
            for(j = i; j > 0 && arr[j-1] > tmp; j--) {
                arr[j] = arr[j-1]; //若当前待比较元素小于被比较元素，则序列往后移位。
            }
            arr[j] = tmp;
        }
    }
}

    另一种插入排序的算法实现：

    ①从第0个元素开始，依次取出元素与当前元素前面的有序序列依次对比。

    ②若当前比较元素小于（大于）有序序列里的被比较元素，则两两交换位置。

    function insertSort(arr) {
    let n = arr.length;
    for(let i = 0; i < n; i++) { //第一层：遍历整个数组，即 依次取出待比较元素。        
        for(let j = i; j > 0; j--) { //第二层：当前待比较元素 与 已排好序列依次对比。
            if(arr[j-1] > arr[j]) {
                [arr[j-1],arr[j]] = [arr[j],arr[j-1]] //若当前待比较元素小于被比较元素，则两两交换位置。
            }
        }
    }
}

    # 复杂度分析

    • 时间复杂度：O(n²)

    与冒泡、选择算法类似，只是由原来的方法体执行次数 (n-1)+(n-2)+......+1 变成了 1+2+...+n 次，即 n * (n - 1) / 2 次。

    • 空间复杂度：O(1)

    算法所需空间资源没有随程序执行次数增加而增大。

    • 插入排序是稳定的

    插入排序和冒泡一样，并不会改变原有的元素之间的顺序，如果遇见一个与插入元素相等的，那么把待插入的元素放在相等元素的后面。所以，相等元素的前后顺序没有改变，从原无序序列出去的顺序仍是排好序后的顺序。

    # 总结

    最坏情况下，此算法移动元素数量大，特别是在元素数量特别多的情况下，性能损耗严重。又因该算法有稳定、快的优点，所以元素数量少的适宜选择此算法排序。
