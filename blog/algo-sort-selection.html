    # 选择排序

    2019.03.13

    顾名思义，选择排序即是从待排序的数据中选出最大（或最小）的元素，顺序放在已排好序的序列最后，直到全部数据排序完毕。

    # 执行过程

    步骤：

    ①首趟先认定数组的第一个数为最小（或大）的数，并记录此索引位置。

    ②从数组的每二个数开始往后的每个数，与当前记录索引位置的最小（或大）数依次比较，若比当前记录索引位置的最小数小（或大），则更新记录的索引为当前数的索引，直到比较完每个数。

    ③比较完每个数后，将当前趟数的数 和 记录索引位置的最小（或大）数 交换位置。

    ④如此，执行n-1趟便得结果。

    # js代码实现

    function selectionSort(arr) {
    let n = arr.length;
    for( let i = 0; i < n - 1 ; i++ ) {
        let minIdx = i;
        for( let j = i + 1 ; j < n ; j++ ) {            
            if( arr[minIdx] > arr[j] ) {
                minIdx = j;
            }
        }
        [arr[i],arr[minIdx]] = [arr[minIdx],arr[i]]
    }
}

    # 复杂度分析

    • 时间复杂度：O(n²)

    与冒泡算法类似，方法体也要进行 (n-1)+(n-2)+......+1 = n * (n - 1) / 2 次。

    • 空间复杂度：O(1)

    算法所需空间资源没有随程序执行次数增加而增大。

    • 是不稳定的排序算法。

    # 总结

    与冒泡排序对比，每趟排序，选择排序最多仅做一对位置交换，而冒泡排序最多可能要做n/2对数据交换，所以这种算法是有利于计算机性能的。但选择排序和冒泡排序有着同样的复杂度。所以就刚才提到的一个优点，二者选其一我选选择排序。
