    # 一台电脑上使用多个 SSH Key

    2020.02.17

    有时需要在家办公，所以家里电脑最初配置sshKey的时候是默认使用公司的邮箱进行配置的。当在学习或者业余工作时也会使用到非公司服务器（如，github、码云等）的git服务器也需要这个sskkey，为了不想和公司的用同一个sshkey，所以我的场景是需要一台电脑上有多个sshkey的存在。

    # 步骤

    # 一、生成 ssh key

    用 ssh-keygen -t rsa -C "你的邮箱地址" 命令生成 ssh key。

    # 二、生成公司 ssh key

    为了方便，公司的git管理服务器我是直接三个回车。

    • 先来了解一下三个回车的意思：
    Generating public/private rsa key pair.
Enter file in which to save the key (/c/Users/qings/.ssh/id_rsa):

    这里回车代表将秘钥默认存储在c盘/Users/用户名/.ssh的文件夹下。

    Enter passphrase (empty for no passphrase):

    这里是设置提交项目的密码（默认是空），用来避免别人用你的电脑修改你的项目后提交。

    Enter same passphrase again:

    再输一次密码。

    # 三、生成 gitHub 上的sshkey

    1. 前往文件夹 /Users/用户名/.ssh，在里面新建github文件夹，用来存放后面生成的密钥。

    2. 出现 Enter file in which to save the key 时输入

    /Users/用户名/.ssh/github/id_rsa

    1. 创建config文件

    在/Users/用户名/.ssh目录下新建config文件，这是为了配置映射功能的，在里面填入下面代码：

    #github的配置
Host github.com
HostName github.com
IdentityFile ~/.ssh/github/id_rsa
User git

    HostName代表服务器名 IdentityFile代表密钥的地址

    config文件中也可以写多个配置：

    如：

    #github配置
Host github.com
HostName github.com
IdentityFile ~/.ssh/github/id_rsa
User git
#gitoschina的配置
Host git.oschina.net
HostName git.oschina.net
IdentityFile ~/.ssh/oschina/id_rsa
User git

    1. 测试

    测试github的SSH key是否设置成功

    终端输入 ssh -t git@github.com

    返回 You've successfully authenticated, but Github does not provide shell access. 即成功的设置了SSH key。
